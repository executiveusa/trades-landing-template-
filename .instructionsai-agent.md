# CLAUDE CODE AGENT RULES: pv-plaster-landing

**You are operating in a jcodemunch-optimized workspace.**

## MANDATORY PROTOCOL

### 1. Index Once (Project-Scoped)
On first interaction, call:
```
index_folder: { "path": "." }
```
Caches the entire project. Reuse for all future queries in this session.

### 2. Search-First Principle
**Before opening ANY file:**
- "Need to find something?" → `search_symbols()`
- "Need exact implementation?" → `get_symbol()`
- "Need file structure?" → `get_file_outline()`
- "Need architecture?" → `get_repo_outline()`

### 3. Token Budget
- Context window is expensive
- Every file read is ~2-5k tokens
- Every jcodemunch query is ~100-500 tokens
- **Spend 15 minutes indexing = save 100k+ tokens over a session**

### 4. Never Violate These Rules
❌ "Read components/Contact.tsx"
✅ "search_symbols(query='Contact')"

❌ "Open lib/tenant.ts, I need to see all exports"
✅ "get_file_outline(file_path='lib/tenant.ts')"

❌ "Show me the first 50 functions in the project"
✅ "search_symbols() and batch with get_symbols()"

## Available Query Patterns

| Pattern | Query | Cost | Use When |
|---------|-------|------|----------|
| Find by name | `search_symbols(query="name")` | 100-300 | Looking for specific function/class |
| Get exact source | `get_symbol(symbol_id="...")` | 150-400 | Need full implementation |
| File structure | `get_file_outline(file_path="...")` | 50-200 | Want exports without loading |
| Architecture | `get_repo_outline()` | 1-2k | Understanding overall design |
| Full-text search | `search_text(query="...")` | 300-500 | No symbol match found |
| Batch retrieval | `get_symbols([...ids])` | 300-800 | Fetching 3+ symbols at once |

## Decision Flow

```
User asks me to work on code...
│
├─ "Update Contact form"
│  └─ search_symbols(query="Contact")
│     → get_symbol(symbol_id="components/Contact.tsx::Contact#function")
│     → Edit & verify
│
├─ "Explain how WhatsApp link works"
│  └─ search_symbols(query="whatsapp")
│     → get_symbol(symbol_id="lib/whatsapp.ts::getWhatsAppLink#function")
│     → Explain
│
├─ "What services are available?"
│  └─ get_symbol(symbol_id="lib/tenant.ts::getTenant#function")
│     → Read tenant.json from cache
│     → Return services array
│
├─ "Where are all the components?"
│  └─ get_repo_outline()
│     → Show high-level structure
│
└─ "Find all CSS variables"
   └─ search_text(query="--brand-")
      → Full-text search for custom properties
```

## Forbidden Patterns

🚫 Opening entire files just to find one function  
🚫 Re-reading the same file multiple times in a session  
🚫 Using file I/O before trying jcodemunch  
🚫 Skipping the index step and calling search multiple times  
🚫 Not batching symbol retrievals with get_symbols()

## Allowed Exceptions

✅ Reading raw JSON/config files (tenant.json)  
✅ Opening binary assets (images in public/)  
✅ Checking .env or secrets (if needed)  
✅ Only if jcodemunch cannot parse the language

- ✅ Faster response times (less context to process)
